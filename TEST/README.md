# **Required Packages**

- VEP (tested with ensembl-vep=110.1)
- plink 2 version alpha 5.11
- Regenie 3.5 above (tested for 3.2.6 but use latest if possible)
- Nextflow (tested with version=25.04.4)

 For plink2 uses system's (ERIS2 cluster's) path so user don't have to install. 
 For VEP user must install the package themselves. User must also install latest version of Regenie.  Regenie/2.0.1 available in OGI cluster do not support SKATO tests.

### Required python packages for Step1 to 7:
- numpy, pandas, seaborn, plotly, matplotlib, scikit-learn, scipy, statsmodels, xlsxwriter, pysnptools
  
  #Use pysnptools (version=0.5.13) for python version 3.9 [pip install pysnptools==0.5.13]

### Required R libraries for QQplot.R
- qqman, ggplot, dplyr, stringr, data.table, optparse

# Pipeline:
 Uses genome build 38 (GRCh38.p13.genome.fa). 
 All outputs are stored in folders of their respective steps: step1, step2, and step7. 

## Pipeline environment:

  Most of the packages are installed in the singularity that the submission file automatically uses. 
  However you will need to create your conda environment to run the singularity and Regenie. 
  Please use the **GeneBurdenCondaEnv.yaml** file to generate the conda environment.

## Parameters/Input Files:
  All the input files needed are listed in the **step1.params.yaml** and **step2.params.yaml** file. Change the path to your input files in this file. You don't need to change the default paramenters.
  
  #### Inputs file formats:
   - Phenotype file must be a tab delimited file and must be formatted as : FID, IID, Phenotype.
   - Covariate file must be a tab delimited file and must be formatted as : FID, IID, No.of.synonymous.Mutations, covar1, covar2, covar3, covar4,
     PC1, PC2, PC3 ...  ##(Important to Note- You don't need the column No.of.synonymous.Mutations here as the nexflow pipeline will compute/add it automatically in your covariate file)
   - Synonymous Counts: As stated above counts of synonymous mutations will be used as a covariates in Regenie. This will be automatically generated by the code. 
   - For very small dataset (n<100 samples)- See jupyter notebook AAF.ipnyb i.e.to generate AAF (only when gnomAD was used in step1 for very small datasets).

   ##### Parameters:

   Check the files **step1.params.yaml** and **step2.params.yaml**. 
   Following are the parameters for step2.

	`
		- AnnotatedVcfName: "path to your annotated vcf" #This is the output from step1 and should be in your work directory under the sub-folder ./step1/annotated.vcf or ./step1/annotated.txt

## Guideline to run Nextflow Pipeline

    1. Input the parameters files - **step1.params.yaml** and **step2.params.yaml**.
    2. sbatch step1.submit.sh
    3. sbatch step2_to_7.submit.sh

## Details on each steps
## STEP 1.
1 a. **sbatch step1.submit.sh**. This step will perform QC (filter missingess, monomorphic varaints, etc) and extract the rare variants.
 Then it will annotate your vcf. 
 
 Following is the input files needed. Please provide the path to the **params.yaml** file. You don't need to change the default options/parameters

   #### Main Arguments (Inputs)
   | command      |                 Description		      |
   |--------------|-------------------------------------------|
   | --vcf        |  Path to your post variant QC vcf file            | 
   | --phenoFile  |  Path to your phenotype file. Your phenotype file must be tab-delimited and have the following columns- <br> 'IID','Phenotype', where IID is the sampleID and Phenotype is your case/control if dichotomous*.| 
   |   ./step1/output5.vcf  |       This input is automatically used by the code and does not need to be specified by the user. <br> This is the output from step 1.a generated in folder step1 of your working directory.     | 
  
   #### Options
   | command                   |                 Description		   |
   |---------------------------|-------------------------------------------|
   | --keep                    |  Path to your file with sample IDs. This allows you to subset only specific samples from full set vcf. <br> If you want to use the full set vcf then you can omit this option. <br> This must be tab-delimited and have the following columns- '#FID', 'IID'.  | 
   | --your_exome_target       |  Path to your bed file. This allows you to filter out off target variants detected in regions of genome not <br> targeted by exome sequencing capture kit. The bed file should be in bed 0 format.| 
   | --missingness  <int>      |  filters out all variants with missing call rates exceeding the provided value (default 0.02) to be removed |
   | --maf  <int>              |   filters out all variants with allele frequency above the provided threshold (default 0.01)  |
   | --plink2_path             |   give the path to plink2 version alpha 5.11. Needed if plink2 is not in path (default: plink2) <br> Use: for ERIS2 plink2_path='/apps/software/plink2/plink2' <br> For OGI plink2_path= /modules/ogi-mbc/software/plink2/alpha5.11/bin/plink2 |
   | --plink3_path             |   give the path to plink2 version alpha 3. This is optional if you encounter error with plink2 version alpha 5 (default: plink3)  |
   
   The pipeline will automatically calculate the allele frequency using your study population instead of Gnomad. It will also automatically filter out varaints in the mitochondrial DNA. 
   The code will automatically compute and filter out mono-morphic variants
